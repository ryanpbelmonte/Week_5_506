Trail Checker

Trail Checker helps hikers quickly decide whether a trail is reasonable to visit by combining location-based weather data with saved user trail checks. The app is designed for people who want a fast, practical trail-planning snapshot before heading outside.

1. Project Name and Pitch

Trail Checker is a web app for hikers and outdoor planners who want a quick trail-readiness check before choosing a hike. Users can search for a trail or location, view current and forecast weather conditions, and save trail checks to their account for later planning.

3. Project Type

Trail Checker. This project is a location-based outdoor planning app. It is similar to a dashboard/tracker project because users can log in, search an external data source, save trail checks, and review previously saved planning information.

4. Endpoints Planned
  • GET / — Show the home page and introduce Trail Checker.
  • GET /trails — Show the trail search page and the logged-in user's saved trail checks.
  • POST /trails/search — Submit a trail/location search and request weather data from the external API.
  • POST /trails/save — Save a searched trail check to the logged-in user's account.
  • GET /trails/<id> — Show one saved trail check with weather details and notes.
  • POST /trails/<id>/delete — Delete one saved trail check from the logged-in user's account.
  • GET /about — Show this shared team project plan.
5. Web-Based Data Source

API: Open-Meteo Weather Forecast API

Docs: https://open-meteo.com/en/docs

Specific endpoint planned: 

https://api.open-meteo.com/v1/forecast?latitude={latitude}&longitude={longitude}¤t=temperature_2m,precipitation,weather_code,wind_speed_10m&daily=temperature_2m_max,temperature_2m_min,precipitation_probability_max,wind_speed_10m_max&timezone=auto

Optional supporting endpoint for location search: 

https://geocoding-api.open-meteo.com/v1/search?name={search_term}&count=5&language=en&format=json

Free-tier notes: Open-Meteo offers free access for non-commercial use and does not require an API key. Its free-use terms list limits of less than 10,000 API calls per day, 5,000 per hour, and 600 per minute.

Backup plan: If the API integration takes longer than expected, the MVP can use a small hard-coded list of sample trail locations with stored latitude and longitude values while still saving trail checks in Postgres. This keeps the login, database, saved-check workflow, and templates working while the API integration is finished.
